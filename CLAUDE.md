# CLAUDE.md — Project Instructions

## Project Overview
This is a **client-acquisition website** designed to showcase services and convert visitors into paying clients.

## Goals
- Present services clearly and professionally
- Build trust through portfolio, testimonials, and credibility signals
- Drive visitors to take action (contact, book a call, hire)

## File Structure
```
project/
├── index.html          # Main website (single-page)
├── CLAUDE.md           # This file — project instructions for Claude
├── SKILLS.md           # Skills and capabilities reference
└── assets/             # Images, icons (optional)
```

## Tech Stack
- **HTML5** — semantic structure
- **CSS3** — custom properties, animations, responsive grid
- **Vanilla JS** — smooth scroll, form handling, interactivity
- No frameworks required (keeps it fast and portable)

## Design Principles
1. **First impression wins** — hero section must instantly communicate value
2. **Social proof matters** — show testimonials, numbers, results
3. **Clear CTA everywhere** — every section guides toward contact
4. **Mobile-first** — most clients browse on phones
5. **Fast loading** — no heavy libraries or bloated code

## Sections to Include
1. **Hero** — headline, subheadline, CTA button
2. **Services** — what you offer, clearly described
3. **About** — who you are, why trust you
4. **Portfolio / Results** — proof of work (even mock examples)
5. **Testimonials** — social proof
6. **Pricing** (optional) — packages or "starting at" ranges
7. **Contact** — form + email + social links
8. **Footer** — copyright, links

## Copywriting Rules
- Use "you/your" more than "I/me" — focus on client benefits
- Lead with outcomes, not process ("You'll get X" not "I do Y")
- Keep sentences short and punchy
- Every section should answer: "Why should I hire this person?"
